## Ответы на вопросы
1. Event Propagation это процесс распространения событий через DOM. Основные фазы включают фазу захвата (событие идет сверху вниз к конкретному элементу), фазу цели и фазу всплытия. На практике можно использовать это для делегирования событий дочерним элементом. Это упрощает код (позволяет повесить обработчик на родителя, а не на каждый элемент отдельно), также данный делегирование корректно работает с элементами, добавленными после изначального формирования DOM-дерева (например, через js). Также можно останавливать распространение с помощью e.stopPropagation()

2. Promise это объект для работы с асинхронными операциями. Его основные состояния: pending (ожидание), fulfilled (выполнено), rejected (ошибка). Пример из тестового:
```getMovies()
      .then(data => setMovies(data))
      .catch(err => setError(err.message))
      .finally(() => setLoading(false))
```
* Над промисами существует синтаксичекий сахар в виде async/await.
* Event loop необходим для реализации асинхронности в js, т.к. базовый js однопоточный.Он ставит асинхронные операции в очередь, затем проверяет стек вызовов и выполняет их, когда стек пуст

3. Объектно-ориентированное программирование это одна из парадигм программирования, где основными сущностями являются объекты. В js реализовано через прототипы. Классы появились только в стандарте ES6. Основные принципы ООП включают в себя инкапсуляцию, наследование и полиморфизм. Пример:
```
class Rectangle {
	constructor(height, width) {
		this.height = height; // Инкапсуляция
		this.width = width;
	}
	draw() {
		console.log("Рисуем прямоугольник");
	}
}

class ColoredRectangleWithText extends Rectangle { // Наследование
	constructor(height, width, text, bgColor) {
		super(height, width);
		this.text = text;
		this.bgColor = bgColor;
	}
	draw() { // Полиморфизм
		console.log("Рисуем прямоугольник c текстом");
	}
}
```

4. После ввода URL в адресную строку, браузер сначала пытается найти его в своем DNS кэше, затем кэше операционной системы и т.д. по DNS серверам. Затем устанавливает соединение и делает http(s) запрос. Если сайт отвечает, то принимает HTML и начинает формировать DOM, CSSOM деревья. Затем рендерит и отрисовывает элементы. Паралелльно (или после, в зависимости от способа подключения скриптов) выполняет JS
* Технологии ускорения: lazy loading, CDN, кэширование
* Проблемы: XSS, CORS ограничения
